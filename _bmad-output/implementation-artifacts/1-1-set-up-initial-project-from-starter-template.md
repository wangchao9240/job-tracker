# Story 1.1: Set Up Initial Project from Starter Template

Status: done

<!-- Note: Validation is optional. Run validate-create-story for quality check before dev-story. -->

## Story

As a developer,
I want to bootstrap the initial Next.js project from the selected starter template and baseline tooling,
so that the team has a working foundation to implement user-facing features safely and consistently.

## Acceptance Criteria

1. Given the repository is ready for initial bootstrap, when the project is created using `create-next-app` with JavaScript (App Router) and Tailwind, then the app starts successfully in development and the default page renders without runtime errors.
2. Given the baseline UI foundation is required, when shadcn/ui is installed and configured for the project, then a minimal shadcn/ui component can be rendered in the app and styling works as expected with Tailwind.
3. Given the project will be deployed later, when the app is built in production mode, then the build completes successfully and ESLint checks (as configured by Next.js) run without failing the build.

## Tasks / Subtasks

- [x] Bootstrap Next.js app from the architecture-selected starter command.
  - [x] Confirm local Node.js is `>= 20.9.0` (required by `create-next-app@latest` for Next 16.1.1).
  - [x] From repo root, run:
    - `npx create-next-app@latest job-tracker --js --app --tailwind --eslint --src-dir --import-alias "@/*" --use-npm`
  - [x] Work inside the generated app directory: `cd job-tracker`.
  - [x] Start dev server and confirm the default page renders without runtime errors:
    - `npm run dev`
- [x] Initialize shadcn/ui and prove Tailwind styling.
  - [x] In `./job-tracker/`, run `npx shadcn@latest init` and choose JavaScript + Tailwind + `src/` layout.
  - [x] Add at least one shadcn/ui component (e.g. `button`) and render it on the default page to confirm styling works.
- [x] Confirm production build + lint.
  - [x] In `./job-tracker/`, run `npm run build` and confirm the build succeeds.
  - [x] In `./job-tracker/`, run `npm run lint` and confirm it passes.

## Dev Notes

### Scope

This story is only about creating a clean, runnable Next.js foundation + baseline UI library wiring.

Out of scope for this story (do not start early):
- Supabase project setup, Auth wiring, or any DB schema/migrations
- Any app feature pages beyond the default landing page
- Any new API endpoints under `src/app/api/**`

### Developer Guardrails (Must-Follow)

From `_bmad-output/project-context.md`:
- JavaScript only (`.js/.jsx`), no TypeScript files or tooling.
- ESM only (`import/export`), no `require()` / `module.exports`.
- Next.js App Router only; `src/app/**` structure.
- Use Tailwind CSS + shadcn/ui; do not add an alternative UI framework.
- Do not add Prettier; keep ESLint only.

From `_bmad-output/architecture.md` (Selected Starter: Option A):
- Use the provided `create-next-app@latest ... --js --app --tailwind --eslint --src-dir --import-alias \"@/*\" --use-npm` command.
- Node.js must be `>= 20.9.0` for `create-next-app@latest`.

### Implementation Notes

- Repository root contains planning artifacts (`_bmad/`, `_bmad-output/`). Keep them as-is.
- The application codebase is created under `./job-tracker/` (per architecture handoff). After bootstrap, run all app commands from `./job-tracker/`.
- Verify shadcn/ui can render a component on the default route (edit `./job-tracker/src/app/page.jsx`).

### Technical Requirements (Versions are project-pinned)

- Runtime: Node.js `>= 20.9.0`.
- Framework: Next.js (App Router) `16.1.1`.
- Styling: Tailwind CSS `4.1.18`.
- UI components: shadcn/ui (installed via `npx shadcn@latest init`).

### Architecture Compliance Checklist

- App Router only: ensure `./job-tracker/src/app/**` exists; do not introduce `pages/`.
- Import alias `@/*`: use it for intra-app imports once app code exists.
- Keep security boundaries in mind early: server-only code belongs under `src/lib/server/**` (future stories); do not place secrets in client code.

### Library & Tooling Requirements

- Package manager: npm (per architecture starter command).
- Linting: keep Next.js default ESLint setup for now; the AC requires `npm run lint` passes.
- Avoid early dependency creep: do not add Supabase packages, AI gateway code, or testing frameworks in this story unless required to satisfy acceptance criteria.

### Testing Requirements

- No Jest setup required for this story; keep scope strictly to bootstrap + lint + build.
- Do not add networked tests or any CI configuration changes as part of this bootstrap story.

### Project Structure Notes

- App root (generated by `create-next-app`): `./job-tracker/`
- Primary route to confirm rendering: `./job-tracker/src/app/page.jsx`
- shadcn/ui components (after init): `./job-tracker/src/components/ui/**`
- shadcn config: `./job-tracker/components.json`
